# Отчёт по проекту: Breedi Genomic Prediction API

## 1. Обзор

**Цель** — REST API сервис для расчёта геномной оценки племенной ценности (**GEBV**, Genomic Estimated Breeding Value) по ДНК-данным животных. Сервис принимает генотипические данные (SNP-дозировки) и возвращает численную оценку генетического потенциала животного.

**Задача** — предоставить быстрый, масштабируемый и документированный API-интерфейс, пригодный для интеграции с фронтендом, мобильным приложением или аналитическими пайплайнами селекции.

**Модель** — **RR-BLUP** (Ridge Regression Best Linear Unbiased Prediction). Математически эквивалентна **GBLUP** и оценивает аддитивный эффект каждого SNP-маркера независимо.

---

## 2. Методология

### 2.1 Модель RR-BLUP

RR-BLUP — метод оценки эффектов SNP-маркеров через штрафную регрессию (L2-регуляризация, аналог Ridge Regression). Модель предполагает, что все SNP-маркеры вносят случайный вклад с общей дисперсией, что соответствует инфинитезимальной модели наследственности.

Математически:

$$\hat{y} = X\beta = \sum_{i=1}^{n} \beta_i \cdot x_i$$

где:
- **βᵢ** — аддитивный эффект *i*-го SNP (обученный на фенотипах + генотипах референсной популяции)
- **xᵢ** — дозировка аллеля для данного животного в локусе *i*
- **n** — число SNP-маркеров

### 2.2 Кодирование генотипов

Входные данные — аддитивные дозировки аллелей:

| Значение | Генотип | Интерпретация |
|---|---|---|
| 0 | Гомозигота по референсному аллелю | Альтернативный аллель отсутствует |
| 1 | Гетерозигота | Один альтернативный аллель |
| 2 | Гомозигота по альтернативному аллелю | Два альтернативных аллеля |

Это стандартное кодирование в современных геномных пайплайнах (PLINK, rrBLUP, GCTA).

### 2.3 Расчёт процентиля

Percentile определяется через CDF стандартного нормального распределения:

$$\text{Percentile} = \Phi\!\left(\frac{\text{GEBV} - \mu_{\text{ref}}}{\sigma_{\text{ref}}}\right) \times 100\%$$

где μ_ref и σ_ref — среднее и стандартное отклонение GEBV референсной популяции, Φ — функция ошибок (error function).

### 2.4 Эквивалентность RR-BLUP ↔ GBLUP

RR-BLUP и GBLUP дают идентичные предсказания, но оперируют в разных пространствах:
- **RR-BLUP**: работает в пространстве маркеров (вектор β ∈ ℝⁿ)
- **GBLUP**: работает в пространстве животных (матрица родства G = ZZᵀ)

Для inference выгоднее RR-BLUP — скалярное произведение O(n), тогда как GBLUP требует обращения матрицы O(n³).

## 3. Входные данные

### 3.1 SNP-дозировки (основной вход)

| Параметр | Значение |
|---|---|
| Поле | `snp_dosages` |
| Тип | `list[float]` |
| Допустимые значения | `{0, 1, 2}` |
| Длина | Должна совпадать с числом SNP в модели |
| Валидация | Pydantic `field_validator` — отклоняет всё, кроме 0/1/2 |

### 3.2 Идентификатор животного

| Параметр | Значение |
|---|---|
| Поле | `animal_id` |
| Тип | `str` |
| Ограничения | 1–64 символа |

### 3.3 Файл модели (`model.joblib`)

Ожидаемый формат — `joblib.dump(dict)` со следующими ключами:

```python
{
    "snp_effects": np.ndarray,   # shape (n_snps,) — эффекты SNP
    "accuracy": float,           # точность на валидации (0–1)
    "ref_mean": float,           # среднее GEBV референсной популяции
    "ref_std": float,            # стандартное отклонение GEBV
    "version": str,              # опционально: версия модели
}
```

Допускается также «legacy» формат — голый numpy-массив `snp_effects` (accuracy = 0.72 по умолчанию).

## 4. Выходные данные

| Поле | Тип | Описание |
|---|---|---|
| `animal_id` | `str` | Идентификатор из запроса |
| `gebv` | `float` | Genomic Estimated Breeding Value — сумма взвешенных SNP-эффектов |
| `accuracy` | `float` | Оценка точности (correlation GEBV ↔ истинная племенная ценность), диапазон [0, 1] |
| `percentile` | `float` | Ранг животного в референсной популяции, диапазон [0, 100] |

**Пример ответа:**

```json
{
  "animal_id": "cow_123",
  "gebv": 1.3184,
  "accuracy": 0.72,
  "percentile": 90.56
}
```

## 5. Архитектура системы

```
┌──────────────┐      POST /predict-gev      ┌──────────────┐
│   Клиент     │ ──────────────────────────► │  FastAPI     │
│ (браузер,    │                             │  (app/main)  │
│  curl, SDK)  │ ◄────────────────────────── │              │
└──────────────┘     JSON-ответ              └──────┬───────┘
                                                    │
                                          ┌─────────▼─────────┐
                                          │  Pydantic         │
                                          │  (валидация 0/1/2)│
                                          └─────────┬─────────┘
                                                    │
                                          ┌─────────▼─────────┐
                                          │  RRBLUPModel      │
                                          │  (inference)      │
                                          │  GEBV = dot(β, x) │
                                          └─────────┬─────────┘
                                                    │
                                          ┌─────────▼─────────┐
                                          │  model.joblib     │
                                          │  app/assets/      │
                                          └───────────────────┘
```

**Ключевые компоненты:**

| Компонент | Файл | Роль |
|---|---|---|
| Приложение | `app/main.py` | FastAPI app, lifespan-хук загрузки модели, роуты |
| Модели данных | `app/models.py` | Pydantic-схемы `PredictRequest`, `PredictResponse`, `HealthResponse` |
| Инференс | `app/inference.py` | Класс `RRBLUPModel` — загрузка `model.joblib`, расчёт GEBV и percentile |
| Модель | `app/assets/model.joblib` | Сериализованные веса SNP-эффектов |
| Скрипт генерации | `scripts/generate_dummy_model.py` | Создание dummy-модели для разработки/тестов |
